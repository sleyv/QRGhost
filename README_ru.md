### 📖 Обзор

**QRGhost** — это безопасный генератор QR-кодов и инструмент шифрования данных с упором на правдоподобное отрицание. Используя архитектуру вложенных скрытых томов в стиле TrueCrypt («луковичное шифрование»), он позволяет пользователям шифровать до 3 уровней секретов в одном QR-коде. Приложение идеально заполняет внутренние секреты шумом CSPRNG до максимальной емкости версии QR-кода, что делает математически невозможным доказательство существования скрытых внутренних уровней без точных паролей.

---

### ✨ Ключевые особенности

- 🔒 **Абсолютное правдоподобное отрицание:** Храните "стрессовый" пароль и реальный секрет. Выдача внешнего пароля раскрывает стрессовый секрет, в то время как реальный секрет остается идеально замаскированным под криптографически безопасный случайный шум.
- 🧅 **Вложенные скрытые тома:** Шифрование данных до 3 вложенных уровней. Вы должны расшифровать внешний слой, чтобы получить доступ к зашифрованному тексту внутреннего слоя. Математически невозможно пропустить или обойти внешний слой, чтобы достичь внутреннего секрета, не указав внешний пароль.
- 📱 **Нативные бинарные QR-коды:** Кодирует данные в режиме "сырых" 8-битных байтов вместо Base64, максимизируя емкость хранения QR-кода.
- 🌊 **Умное распределение емкости:** Автоматически выбирает лучшую версию QR-кода и использует алгоритм "наполнения водой" для безопасного усечения слишком большого текста, чтобы он точно соответствовал оставшейся емкости.
- 📋 **Интеграция с буфером обмена:** Быстрая расшифровка путем вставки (Ctrl+V) изображения QR-кода прямо на вкладку "Decrypt" (Расшифровать).
- 📥 **Самодостаточность:** Скачайте один переносимый файл `qrghost.html` для легкого и безопасного обмена и использования в автономном режиме без подключения к сети.
- 🖥️ **Поддержка PWA:** Работает в автономном режиме после первого посещения сайта.
- 🚀 **Несколько вариантов развертывания:** как отдельный файл, через Docker или за обратным прокси.
- 🌐 **Гибкий хостинг:** Может размещаться с использованием любого веб-сервера (Apache, Nginx и т.д.).

### 🚀 С чего начать

#### 1. Клонировать и подготовить
```bash
git clone https://github.com/sleyv/QRGhost.git
cd QRGhost
```

#### 2. Выбрать свой метод

<details>
<summary><b>📥 Вариант 1: Запуск в виде локального файла (Самый простой)</b></summary>

Вы можете использовать QRGhost без какой-либо установки.
1. Скачайте файл `qrghost.html` с сайта [Live Demo](https://qrghost.sley.nl), нажав на кнопку `📥 HTML`.
2. В качестве альтернативы используйте файл `index.html` из этого репозитория.
3. Откройте файл прямо в своем веб-браузере.
</details>

<details>
<summary><b>💻 Вариант 2: Локальный Сервер</b></summary>

Вы можете запустить простой веб-сервер прямо на своем компьютере. SSL-сертификаты не требуются для localhost.

1. **Запустите локальный статический сервер** (например, встроенный http.server в Python):
   ```bash
   python3 -m http.server 8222
   ```
2. Откройте приложение по адресу [http://localhost:8222](http://localhost:8222).
</details>

<details>
<summary><b>🐳 Вариант 3: Docker (Автономный)</b></summary>

Docker не требует SSL-сертификатов для локальной разработки. SSL необходим только при размещении в Интернете.

1. **Соберите и запустите контейнер**:
   ```bash
   docker-compose up -d --build
   ```
2. Откройте приложение по адресу [http://localhost:8222](http://localhost:8222).
</details>

<details>
<summary><b>🌐 Вариант 4: Обратный прокси (Рекомендуется для продакшена)</b></summary>

Это рекомендуемый метод для размещения в сети. Обратный прокси-сервер (такой как Nginx или Traefik) будет обрабатывать HTTPS.

Прилагаемый файл `docker-compose-traefik.yml` является **примером** и должен быть адаптирован к вашей конфигурации.

1. Переименуйте `docker-compose-traefik.yml` в `docker-compose.yml` или используйте флаг `-f`.
2. Измените файл так, чтобы он соответствовал вашей конфигурации Traefik (например, правило `Host` и имя внешней сети).
3. **Соберите и запустите контейнер**:
   ```bash
   docker-compose -f docker-compose-traefik.yml up -d --build
   ```
</details>

<details>
<summary><b>🖥️ Вариант 5: Хостинг с любым веб-сервером</b></summary>

Файл `index.html` можно разместить на любом веб-сервере или сервисе хостинга статических файлов, поддерживающем HTTPS.

1. Загрузите статические файлы (`index.html`, `manifest.json`, `sw.js`, иконки) на свой веб-сервер.
2. Настройте HTTPS, используя предпочитаемый вами метод (Let's Encrypt, Cloudflare и т.д.).
3. Получите доступ к приложению через свой домен по протоколу HTTPS.
</details>

---

### ⚠️ Важные замечания

- **HTTPS обязателен** для корректной работы всех функций браузера при размещении в Интернете. Приложение не будет работать по протоколу HTTP. Исключением является доступ к localhost.
- Вы можете разместить HTML-файл на любом сервисе веб-хостинга или библиотеке, которая поддерживает статические файлы и HTTPS.
